SECTION 4 — BROWSER AUTOMATION SPECS
================================================================================

BROWSER: Brave on Windows
BRAVE PATH: C:\Program Files\BraveSoftware\Brave-Browser\Application\brave.exe
REMOTE DEBUG PORT: 9222
TELEGRAM URL: https://web.telegram.org

CONNECTION METHOD:
- Use Playwright's connect_over_cdp() to connect to existing Brave session
- Do NOT launch a new browser — connect to the already-running one
- If connection fails, print clear instructions and exit gracefully

FINDING TELEGRAM TAB:
- Iterate through all browser contexts and pages
- Find the page whose URL contains "web.telegram.org"
- If not found, instruct user to open Telegram Web and restart

DETECTING UNREAD MESSAGES:
- Every CHECK_INTERVAL seconds, scan the chat list
- Look for chats with unread message indicators
- Use multiple CSS selector fallbacks since Telegram Web updates its selectors:
  Primary:   ".chatlist-chat:has(.badge-unread)"
  Fallback1: "[data-peer-id]:has(.unread-count)"
  Fallback2: ".chat-item:has(.badge)"
- If selectors fail, log a warning but don't crash

READING MESSAGES:
- When an unread chat is found, click it and wait 1.5s for messages to load
- Find the last incoming message (not sent by us — class "is-out" = our message)
- Extract the message text
- Extract a unique message ID (data-mid attribute) to avoid re-replying
- Skip if message ID already exists in replied_messages set
- Skip if message text is empty or only contains media/stickers (no text)

TYPING & SENDING:
- Find the message input box using multiple selector fallbacks:
  Primary:   ".input-message-input"
  Fallback1: "[contenteditable='true']"
  Fallback2: ".composer-input"
- Type character by character with randomized delays (simulate human typing)
- Base typing speed: 0.04s per character + random(0, 0.03)
- Wait random(0.5, 1.5) seconds after typing before pressing Enter
- Press Enter to send
- After sending, wait random(1, 3) seconds before moving to next chat
